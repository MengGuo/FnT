\section{Task Relaxation and Adaptation in Unknown Workspaces}
\label{ch03:sec:task_relaxation_adaptation}

The synthesis framework in Chapter~\ref{chap:preliminaries} presumes that the
weighted transition system is available before execution and remains fixed.
This assumption is restrictive in partially known workspaces, where blocked
passages, newly discovered traversable regions, or revised semantic labels may
appear only after the robot starts moving. Under such uncertainty, a local LTL
specification may be infeasible over the current abstraction even when the
physical task is achievable in the actual workspace. At the same time,
safety-critical requirements must remain enforceable throughout execution.

This section addresses that setting for a single robot. The local task is split
into a hard component, which must always be satisfied, and a soft component,
which should be satisfied to the largest possible extent. The first part of the
section develops a relaxed synthesis mechanism for the case in which exact
satisfaction is not currently available. The second part introduces a
history-aware adaptation rule that revises the remaining plan whenever the
workspace abstraction is updated. The construction is related to specification
relaxation and automata-based replanning
\cite{kim2012approximate,guo2013revising,livingston2012backtracking}, but is
presented here in notation consistent with
Chapter~\ref{chap:preliminaries}.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\subsection{Infeasible tasks}\label{ch03:sec:infeasible}
We follow the same notation as in Chapter~\ref{chap:preliminaries}. The initial task~$\varphi$ assigned to the robot might be infeasible by Definition~\ref{ch02:def:feasible}, e.g., either the task is actually infeasible or the task is feasible but the initial workspace model~$\mathcal{T}_c$ is incomplete or incorrect.
The nominal solution proposed in Chapter~\ref{chap:preliminaries} would simply returns failure. An intriguing question to ask is as follows:

\begin{problem}
Assume the task specification is \textbf{infeasible}, how should the specification be \textbf{relaxed} and more importantly how to synthesize the discrete plan that satisfies the original specification \textbf{as much as possible}?
\end{problem}


%On the other hand, this work complements the topic about revising the motion plan under fixed LTL specifications when the workspace model or robot dynamics are updated, like in the cases of real-time revising~\citep{guo2013revising} and local ``patching''~\cite{livingston2012backtracking}.

The work by~\cite{kim2012approximate} partially answers the above problem. It generates a relaxed specification automaton $\mathcal{A}'_{\varphi}$ which is close to $\mathcal{A}_{\varphi}$ and feasible over $\mathcal{T}_c$, see Section~III-C of~\cite{kim2012approximate}. Then the discrete plan can be synthesized by following the   procedure as described in Section~\ref{ch02:sec:solution}.  However there are often more than one accepting run within $\mathcal{T}_c\otimes \mathcal{A}'_{\varphi}$ and they may fulfill the original specification to different extents. Instead we aim to firstly find the discrete plan that fulfills the task \emph{the most} by certain criterion, based on which then the specification automaton is relaxed.

\subsubsection{Relaxed product automaton}\label{ch03:subsec:relaxed-prod}

By Definition~\ref{ch02:def:feasible},  $\varphi$ is infeasible when the standard PBA $\mathcal{A}_p$ does not have an accepting run. Thus we need to relax the constraints imposed by $\mathcal{A}_{\varphi}$ to allow more transitions within $\mathcal{A}_p$, by the relaxed PBA below.

\begin{definition} \label{ch03:def:relaxpro}
The {relaxed} PBA $\mathcal{A}_r=\mathcal{T}_c  {\times} \mathcal{A}_{\varphi}=(Q',\, 2^{AP},\, \delta', \, Q_0',\, \mathcal{F}',\, W_r)$ is defined as:
$Q'=\Pi \times Q= \{\langle \pi,\, q \rangle \, |\, \forall \pi\in \Pi,\, \forall q \in Q\}$;
$2^{AP}$ is the alphabet: $AP=\{a_1,\, a_2,\cdots,\, a_K\}$;
$\delta': Q' \rightarrow 2^{Q'}$. $\langle \pi_j,\, q_n \rangle \in \delta'(\langle \pi_i,\, q_m \rangle)$ if and only if  $(\pi_i, \, \pi_j)\in \longrightarrow_c$ and $\exists \, l \in 2^{AP}$ such that $q_n \in \delta(q_m,\, l)$;
$Q_0'=\Pi_0 \times Q_0$ and $\mathcal{F}'=\Pi \times \mathcal{F}$ are the initial and accepting states;
$W_r: Q'\times Q' \rightarrow \mathbb{R}^+$is the weight function to be defined. \hfill $\blacktriangle$
\end{definition}

Two differences between $\mathcal{A}_r$ and $\mathcal{A}_p$ from Definition~\ref{ch02:def:product} are: (i) the constraint ``$q_n \in \delta(q_m,\,  L_c(\pi_i))$'' when defining $\delta'$ is relaxed to ``$\exists \, l \in 2^{AP}$ such that $q_n\in \delta(q_m,\,  l)$'' here; (ii) the weight function $W_r$ is defined differently from $W_p$. We firstly introduce the evaluation function $\texttt{Eval}: 2^{AP}\rightarrow \{0,\, 1\}^K$:
\begin{equation}\label{ch03:evaluate}
\texttt{Eval}(l)=\mathbf{\nu}\Longleftrightarrow [\, \mathbf{\nu}_i \, ] =
\begin{cases}
1 & \qquad \text{if $a_i\in l$}, \\
0 & \qquad \text{if $a_i\notin l$},
\end{cases}
\end{equation}
where $i=1,\cdots,K$; $l \in 2^{AP}$ and $\mathbf{\nu}\in \{0,\, 1\}^K$.  Namely, each subset of $2^{AP}$ is mapped to a $K$-dimensional Boolean vector. Then a distance function between two input alphabets $\rho:2^{AP}\times 2^{AP}\rightarrow \mathbb{N}$ is defined as:
\begin{equation}\label{ch03:dist}
\rho(l,\,l')=\|\mathbf{\nu}-\mathbf{\nu}'\|_1=\sum_{i=1}^{K}|\, \mathbf{\nu}_i-\mathbf{\nu}'_i\, |,
\end{equation}
where  $\mathbf{\nu}=\texttt{Eval}(l)$, $\mathbf{\nu}'=\texttt{Eval}(l')$ and $l,\, l'\in 2^{AP}$. $\|\cdot\|_1$ is the $\ell_1$ norm. Then we could   define the distance between an element $l\in 2^{AP} $ to a set $\chi \subseteq 2^{AP}$($\chi \neq \emptyset$)~\citep{boyd2004convex}:
\begin{equation}\label{ch03:eq:setdist}
 \texttt{Dist}(l, \, \chi)=\begin{cases}
%\quad -\min_{l\in (2^{\Pi} \backslash \omega)}\; \rho(L(q'),\, l) & \text{if} \quad  L(q') \in \omega \\
\quad 0 & \text{if} \quad  l \in \chi, \\
\quad \min_{l'\in \chi}\; \rho(l,\,  l')& \text{otherwise}.
\end{cases}
\end{equation}

Note that $\texttt{Dist}(l, \, \chi)$ is not defined for $\chi=\emptyset$. An example of computing  $\texttt{Dist}(\cdot)$  is given in Figure~\ref{ch03:distexam}. Now we give the formal definition of $W_r$:
\begin{equation}\label{ch03:eq:WR}
\begin{split}
&W_r(\langle \pi_i,\, q_m \rangle, \, \langle \pi_j,\, q_n \rangle)\\
&=W_c(\pi_i,\, \pi_j) + \alpha \,\cdot \texttt{Dist}(L_c(\pi_i),\, \chi(q_m,\,q_n)),
\end{split}
\end{equation}
where $\langle \pi_j,\, q_n \rangle \in \delta'(\langle \pi_i,\, q_m \rangle)$; $\alpha\geq 0$ is a design parameter;
\begin{equation}\label{ch03:eq:chi}
\chi(q_m,\,q_n)=\{l\in 2^{AP}| \, q_n\, \in \delta(q_m,\, l) \}
\end{equation}
consists of all input alphabets that enable the transition from $q_m$ to $q_n$ in $\mathcal{A}_{\varphi}$. By Definition~\ref{ch03:def:relaxpro} there always exists $l \in 2^{AP}$   that $ q_n \in \delta(q_m, l)$, thus $\chi(q_m,\, q_n) \neq \emptyset$ is ensured. $W_c(\pi_i,\, \pi_j)$ is the  cost of the transition from $\pi_i$ to $\pi_j$ in $\mathcal{T}_c$; $\texttt{Dist}(L_c(\pi_i),\, \chi(q_m,\,q_n) )$ measures how much the transition from $\pi_i$ to $\pi_j$   violates the constraints imposed by the transition from $q_m$ to $q_n$.
%Being $0$ means that $\mathcal{A}_{\varphi}$ is not violated, while the larger the distance is the more $\mathcal{A}_{\varphi}$ is violated.
The design parameter~$\alpha$ reflects the relative penalty on violating the original specification,  and also the user's {preference} on a plan that has less  cost or that fulfills the task  more.
%The penalty on violating $\mathcal{A}_{\varphi}$ is increased when $\alpha$ is larger.

\begin{example}\label{ch03:examp:dist}
Consider the NBA in Figure~\ref{ch02:fig:Buchi} and the  transition from state $q_1$ to $q_3$, $\chi(q_1,\,q_3)=\{\{a_2\},\, \{a_2, \, a_1\}, \, \{a_2, \, a_3\}, \, \{a_2,\, a_1, \, a_3\}\}$. Then
$\textup{\texttt{Dist}}(\{a_1\}, \chi(q_1,\,q_3))=\rho(\{a_1\}, \, \{a_1,\, a_2\})=1; \;
\textup{\texttt{Dist}}(\{a_2\}, \chi(q_1,\,q_3))=\rho(\{a_2\}, \, \{a_2\})=0;\;
\textup{\texttt{Dist}}(\{a_2,\, a_3\},$
$\chi(q_1,\,q_3))=\rho(\{a_2,\, a_3\}, \, \{a_2, \, a_3\})=0$.
\hfill $\blacktriangle$
\end{example}

\subsubsection{Balanced accepting run}\label{ch03:subsec:probStatement}
Since~$\mathcal{A}_p$ does not have an accepting run, we instead search for an accepting run within $\mathcal{A}_r$. However the existence of an accepting run alone is not enough because:  (i) they have different implementation  costs; (ii) we would like to measure how much they violate the original specification. Thus we still consider the accepting runs with the  \emph{prefix-suffix} structure by~\eqref{ch03:eq:prod-Apath2}:
\begin{equation} \label{ch03:eq:prod-Apath2}
\begin{split}
 R&=q'_0\, q'_1\cdots q'_{f-1}[\,q'_f \, q'_{f+1}\cdots \cdots q_n'\,]^{\omega}\\
&=\langle \pi_0, q_0\rangle\,\cdots \langle \pi_{f-1}, q_{f-1}\rangle\, \left[\, \langle \pi_f, q_f\rangle\langle \pi_{f+1}, q_{f+1}\rangle \cdots \cdots \langle \pi_n, q_n\rangle\, \right]^{\omega}\, ,
\end{split}
\end{equation}
where $q_0'=\langle \pi_0, q_0\rangle \in Q'_{0}$ and $q_f'=\langle \pi_f, q_f\rangle \in \mathcal{F}'$. Its total cost is calculated differently:
\begin{equation}\label{ch03:eq:prod-cost2}
\begin{split}
\texttt{Cost}(R,\, \mathcal{A}_r)
%&=\sum_{i=0}^{f-1} \big(W_{c}(\pi_i,\, \pi_{i+1})+\alpha \cdot \texttt{Dist}(L_c(\pi_i), \chi(q_{i},q_{i+1})) \big)\\
%&\quad +\gamma \sum_{i=f}^{n-1} \big(W_{c}(\pi_i,\, \pi_{i+1})+ \alpha \cdot \,\texttt{Dist}(L_c(\pi_i), \chi(q_{i},q_{i+1}))\big)\\
&=\texttt{cost}_{\tau}+\alpha\cdot \texttt{dist}_{\varphi}\, ,
\end{split}
\end{equation}
where the accumulated implementation cost of the motion plan $\tau=R|_{\Pi}$ is
\begin{equation}\label{ch03:eq:cost}
\texttt{cost}_{\tau}= \big(\sum_{i=0}^{f-1}+\gamma \sum_{i=f}^{n-1}\big)\, W_{c}(\pi_i,\, \pi_{i+1}) ;
\end{equation}
the accumulated distance of $\tau$ to $\mathcal{A}_{\varphi}$ is
\begin{equation}\label{ch03:eq:dist}
\texttt{dist}_{\varphi}= \big(\sum_{i=0}^{f-1}+\gamma \sum_{i=f}^{n-1}\big)\, \texttt{Dist}(L_c(\pi_i), \chi(q_{i},q_{i+1}));
\end{equation}
the design parameter $\gamma\geq 0$ represents the relative weighting on the cost of transient response (the prefix) and steady response (the suffix).
\begin{problem}\label{ch03:pro:balanced}
Find the accepting run of $\mathcal{A}_r$ that {minimizes} the cost by~\eqref{ch03:eq:prod-cost2}. \hfill $\blacktriangle$
\end{problem}

We call the solution to Problem~\ref{ch03:pro:balanced} the \emph{balanced} accepting run of $\mathcal{A}_r$, denoted by $R_{\text{bal}}$. The corresponding balanced plan is $\tau_{\textup{bal}}=R_{\text{bal}}|_{\Pi}$.
\begin{remark}
As mentioned in Section~\ref{ch02:subsec:NBA}, each transition of the NBA is encoded by a boolean expression accepting all alphabets that enable this transition. This expression can be represented as a binary decision diagram (BBD) such that the distance function by~\eqref{ch03:eq:setdist} can be easily integrated. The idea is that for operator ``$\vee$'' it returns the~\emph{minimal} distance of its left and right branches, while for operator ``$\wedge$'' it returns the~\emph{summed} distance of them. Thus it is not necessary to enumerate all input alphabets to evaluate the distance. More details can be found in Section~\ref{ch02:chapter:experiment}. \hfill $\blacktriangle$
\end{remark}

\begin{lemma}\label{ch03:lem:distanceProof}
If $\textup{\texttt{dist}}_{\varphi}=0$, then $\tau_{\textup{bal}} \models \varphi$.
\end{lemma}
\begin{proof}
Since  $\texttt{Dist}(\cdot)\geq 0$ by~\eqref{ch03:eq:setdist}, the accumulated distance $\texttt{dist}_{\varphi}=0$ implies $q_n\in \delta(q_m,L_c(\pi_i))$ for all transitions $(\langle \pi_i,\, q_m \rangle, \langle \pi_j,\, q_n \rangle )$ along $R_{\text{bal}}$.
Since $\mathcal{A}_p$ and $\mathcal{A}_r$ have the same states with the same sets of initial and accepting states, $R_{ \text{bal}}$ is also an accepting run for $\mathcal{A}_p$ by Definition~\ref{ch02:def:product}. Then its corresponding plan $\tau_{\text{bal}}$ satisfies $\varphi$ by Lemma~\ref{ch02:lem:satisfy}.
\end{proof}

\begin{theorem}
If $\varphi$ is feasible over $\mathcal{T}_c$, the balanced plan $\tau_{\textup{bal}}$ \textbf{satisfies} $\varphi$ if $\alpha> \underline{\alpha}$, where $\underline{\alpha}$ is given by~\eqref{ch03:underalpha}.
\end{theorem}
\begin{proof}
If $\varphi$ is feasible over $\mathcal{T}_c$, Algorithms~\ref{ch02:alg:optimal} return the optimal accepting run $R_{\textup{opt}}$ with the total cost (under the same $\gamma$) by~\eqref{ch02:eq:prod-cost}:
\begin{equation}\label{ch03:underalpha}
\texttt{Cost}(R_{\textup{opt}}, \mathcal{A}_p)=\underline{\alpha}.
\end{equation}
Clearly, $R_{\textup{opt}}$ is also a valid accepting run of ${\mathcal{A}}_r$ since $\mathcal{A}_p$ and $\mathcal{A}_r$ have the same states with the same sets of initial and accepting states. Moreover, under the same $\gamma$, $\texttt{Cost}(R_{\textup{opt}}, \mathcal{A}_p)=\texttt{Cost}(R_{\textup{opt}}, {\mathcal{A}}_r)$.
Assuming that $\tau_{\textup{bal}}$ does not satisfy $\varphi$, then $\texttt{dist}_{\varphi}\geq 1$ by~\eqref{ch03:eq:dist}. As a result,  the total cost of $R_{\textup{bal}}$ by~\eqref{ch03:eq:prod-cost2} satisfies:
$
\texttt{Cost}(R_{\textup{bal}},\, {\mathcal{A}}_r)> \alpha \cdot \texttt{dist}_{\varphi} >\alpha.
$
Since $\alpha>\underline{\alpha}=\texttt{Cost}(R_{\textup{opt}}, \mathcal{A}_p)=\texttt{Cost}(R_{\textup{opt}}, {\mathcal{A}}_r)$, it implies
$
\texttt{Cost}(R_{\textup{bal}}, {\mathcal{A}}_r)>\texttt{Cost}(R_{\textup{opt}}, {\mathcal{A}}_r).
$
However by the definition of the balanced run,~$R_{\textup{bal}}$ is the accepting run of $\mathcal{A}_r$ with the least total cost, i.e.,
$
\texttt{Cost}(R_{\textup{bal}}, {\mathcal{A}}_r)\leq \texttt{Cost}(R_{\textup{opt}}, {\mathcal{A}}_r),
$
which leads to a contradictory. Thus the proposed method can be applied directly when $\varphi$ is feasible over $\mathcal{T}_c$ but choosing~$\alpha>\underline{\alpha}$. Algorithm~\ref{ch02:alg:optimal} will automatically select the accepting run that satisfies $\varphi$, i.e., $\texttt{dist}_{\varphi}=0$.
\end{proof}

\subsection{Soft and hard specifications}\label{ch03:sec:soft-hard}

The previous section presents how to synthesis a balanced motion and task plan that satisfies the potentially-infeasible task as much as possible. However, sometimes a specification contains two distinctive parts: one part for hard constraints that concerns safety or security and should not be violated for all time; another part for soft constraints and additional achievement that should be satisfied as much as possible. In this section, we propose a solution that meets these requirements.
%==========
%==========
\subsubsection{Problem formulation}\label{ch03:subsec:formulation}
The robot's finite transition system is still denoted by $\mathcal{T}_c$ from Definition~\ref{ch02:def:FTS}. Its {task specification} remains an LTL formula over $AP$, but with the following structure:
\begin{equation}\label{ch03:eq:spec}
\varphi=\varphi^{\text{soft}} \, \wedge \, \varphi^{\text{hard}},
\end{equation}
where $\varphi^{\text{soft}}$ and $\varphi^{\text{hard}}$ are ``soft'' and ``hard'' sub-formulas;  $\varphi^{\text{hard}}$ could include safety constraints like  collision avoidance: ``avoid all obstacles'' or power-supply guarantee: ``visit the charging station infinitely often'';
$\varphi^{\text{soft}}$ could include performance requirements like ``collect as many objects'' but the location of some objects is not known. Introducing soft and hard specifications is due to the observation that the partially-known workspace considered in Section~\ref{ch03:sec:partial} might render parts of the specification infeasible initially and thus yield the needs for them to be relaxed, while the safety-critical parts should not be relaxed during the process.

\begin{problem}
Given the task specification by~\eqref{ch03:eq:spec}, how to synthesize the motion and task plan such that $\varphi^{\textup{hard}}$ is satisfied \textbf{fully}, while $\varphi^{\textup{soft}}$ is satisfied \textbf{the most}? \hfill $\blacktriangle$
\end{problem}
%=======

\subsubsection{Safety-ensured product automaton}\label{ch03:subsec:intersection}

Now there are two levels of specifications by $\varphi^{\text{hard}}$ and $\varphi^{\text{soft}}$, with respect to which the property of a plan can be stated more specifically.

%============
\begin{definition}\label{ch03:def:vs}
An finite or infinite path $\tau=\pi_1\pi_2\cdots$ of $\mathcal{T}_c$ is called: (i) \textbf{valid} if $(\pi_i,\, \pi_{i+1})\in \longrightarrow_c$, for $i=1,2,\cdots$; (ii) \textbf{safe} if $\tau \models \varphi^{\textup{hard}}$; (iii) \textbf{satisfying} if $\tau \models \varphi$.     \hfill $\blacktriangle$
\end{definition}

 Let $\mathcal{A}^{\text{hard}}=(Q_1, \,2^{AP},\, \delta_1,\, Q_{1,0},\, \mathcal{F}_1)$ and $\mathcal{A}^{\text{soft}}=(Q_2, \,2^{AP},\, \delta_2,\, Q_{2,0},\, \mathcal{F}_2)$ be the NBA associated with $\varphi^{\text{hard}}$ and $\varphi^{\text{soft}}$, respectively. Detailed definition can be found in Section~\ref{ch02:subsec:NBA}. The functions $\chi_1(\cdot)$ of $\mathcal{A}^{\text{hard}}$ and $\chi_2(\cdot)$ of $\mathcal{A}^{\text{soft}}$  are defined analogously as~\eqref{ch03:eq:chi}. Now we propose a way to construct the relaxed but safety-ensured intersection of $\mathcal{A}^{\text{hard}}$ and $\mathcal{A}^{\text{soft}}$.

%============
\begin{definition}\label{ch03:def:conj}
The relaxed intersection of $\mathcal{A}^{\textup{hard}}$ and $\mathcal{A}^{\textup{soft}}$ is defined by:
\begin{equation}\label{ch03:eq:intersect}
\tilde{\mathcal{A}}_{\varphi}=(Q,\, 2^{AP},\, \delta, \, Q_0,\, \mathcal{F}),
\end{equation}
where $Q=Q_1 \times Q_2 \times \{1,2\}$; $Q_{0}=Q_{1,0} \times Q_{2,0} \times \{1\}$; $\mathcal{F}=\mathcal{F}_{1}\times Q_{2} \times \{1\}$;   $\delta: Q \times 2^{AP} \rightarrow 2^Q$, with $ \langle \check{q}_1,\, \check{q}_2,\, \check{t}\rangle \in \delta(\langle q_1,\, q_2,\, t\rangle, \,l)$ when three conditions hold: (1) $l\in \chi_1(q_1,\,\check{q}_1)$; (2) $\chi_2(q_2,\,\check{q}_2)\neq \emptyset$; (3) $q_{t}\notin \mathcal{F}_{t}$ and $\check{t} =t $, \emph{or} $q_{t}\in \mathcal{F}_{t}$ and $\check{t} =\textup{\texttt{mod}}\, (t ,\, 2)+1$, where $t\in\{1,2\}$ and $\textup{\texttt{mod}}$ is the modulo operation. \hfill $\blacktriangle$
\end{definition}

Algorithm~\ref{ch03:alg:relaxedint} constructs $\tilde{\mathcal{A}}_{\varphi}$ by Definition~\ref{ch03:def:conj}. Note that $\tilde{\mathcal{A}}_{\varphi}$ remains a B\"uchi automaton. We relax the requirement that there should exist a common input alphabet that enables the transitions from $q_i$ to $\check{q}_i$ for $i\in \{1, 2\}$, compared with the standard definition of B\"uchi automata intersection (see Chapter 4.3 of~\cite{baier2008principles}).  An accepting run $r$ of $\tilde{\mathcal{A}}_{\varphi}$ intersects with the accepting set $\mathcal{F}$ infinitely often. The last component $t\in\{1,2\}$ in $Q$ ensures that $r$ has to intersect with both $\mathcal{F}_1\times Q_2 \times \{1\}$ and ${Q}_1\times \mathcal{F}_2 \times \{2\}$. This fact is used in the proof of Theorem~\ref{ch03:theo:saferun} below. Denote by $r|_{Q_1}$ and $r|_{Q_2}$ the projection of $r$ onto the states of $\mathcal{A}^{\text{hard}}$ and $\mathcal{A}^{\text{soft}}$, respectively.

%==========================
\begin{theorem}\label{ch03:theo:saferun}
Given an accepting run $r$ of $\tilde{\mathcal{A}}_{\varphi}$, $r|_{Q_1}$ is an accepting run of $\mathcal{A}^{\textup{hard}}$. Moreover, $\mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi})\subseteq \mathcal{L}_{\omega}(\mathcal{A}^{\textup{hard}})$.
\end{theorem}
\begin{proof}
By definition, at least one of accepting states in $\mathcal{F}$ should appear in $r$ infinitely often. The projection of $\mathcal{F}$ onto $Q_1$ is $\mathcal{F}_1$, therefore one of the accepting states in $\mathcal{F}_1$ is visited infinitely often by $r|_{Q_1}$. Secondly, since $l\in \chi_1(q_1,\,\check{q}_1)$ is ensured by Definition~\ref{ch03:def:conj}, all transitions along $r$ are valid for $\mathcal{A}^{\text{hard}}$. As a result, $r|_{Q_1}$ is an accepting run of $\mathcal{A}^{\text{hard}}$. For the second part, given any infinite word $\sigma\in \mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi})$, $\sigma$ results in an accepting run of $\tilde{\mathcal{A}}_{\varphi}$ by Definition~\ref{ch02:def:cor-run}, denoted by $r_{\sigma}$. It has been proved that $r_{\sigma}|_{Q_1}$ is also an accepting run of  $\mathcal{A}^{\text{hard}}$, which implies that $\sigma \in \mathcal{L}_{\omega}(\mathcal{A}^{\text{hard}})$. Thus for any $\sigma \in \mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi})$, $\sigma \in \mathcal{L}_{\omega}(\mathcal{A}^{\text{hard}})$ holds, namely $\mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi})\subseteq \mathcal{L}_{\omega}(\mathcal{A}^{\text{hard}})$.
%The reason that $\mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi_k})$ is a subset of $\mathcal{L}_{\omega}(\mathcal{A}_k^{\text{hard}})$ is because $\mathcal{A}_k^{\text{soft}}$ might enforce additional constraints.
\end{proof}

Since we need to guarantee that $\varphi^{\text{hard}}$ is fulfilled fully and $\varphi^{\text{soft}}$ is satisfied as much as possible, we rely on the relaxed product automaton proposed previously to handle both feasible and potentially infeasible specifications.


\begin{definition} \label{ch03:def:safe}
The safety-ensured and relaxed product B\"uchi automaton $\tilde{\mathcal{A}}_{r}=\mathcal{T}_c \, {\times}\, \tilde{\mathcal{A}}_{\varphi}=(Q',\, \delta', \, Q_0',\, \mathcal{F}',\, W_p)$ is defined as follows:
$Q'=\Pi \times Q =\{ \langle \pi,\, q \rangle\,|\, \forall \pi\in \Pi,\, \forall q \in Q \}$;
 $\delta': Q' \rightarrow 2^{Q'}$. $\langle \pi_j,\, q_n \rangle\in \delta'(\langle \pi_i,\, q_m \rangle)$ if and only if $(\pi_i,\pi_j)\in \longrightarrow_c$ and $q_n\in \delta(q_m,\, L_c(\pi_i))$;
$Q_{0}'= \Pi_{0}\times Q_0$ and $\mathcal{F}'=\Pi \times \mathcal{F}$ are the initial and  accepting states; $W_r:Q'\times Q'\rightarrow \mathbb{R}^+$ is the weight function:
\begin{equation}\label{ch03:eq:weight-soft}
\begin{split}
 \,&W_r(\langle \pi_{i},\,q_m\rangle, \, \langle \pi_{j},\,q_n\rangle)\\
=\,&W_c(\pi_i, \,  \pi_j)+\alpha \cdot \textup{\texttt{Dist}}(L_c(\pi_i), \, \chi_2(q_2,\, \check{q}_2))
\end{split}
\end{equation}
where $\langle \pi_{j},\,q_n\rangle\in \delta'(\langle \pi_{i},\,q_m\rangle)$; $q_m=\langle q_1,\, q_2, t \rangle$; $q_n=\langle \check{q}_1,\, \check{q}_2, \check{t} \rangle$; $\alpha \geq 0$ is a design parameter; function \textup{\texttt{Dist}($\cdot$)} is defined in~\eqref{ch03:eq:setdist}.  \hfill $\blacktriangle$
\end{definition}

The weight function consists of two parts: $W_c(\pi_i,\,\pi_j)$ measures the implementation cost of the transition from $\pi_i$ to $\pi_j$; $\texttt{Dist}(L_c(\pi_i),\, \chi_2(q_2,\,\check{q}_2))$ measures how much this transition violates the constraints imposed by~$\mathcal{A}^{\text{soft}}$;~$\alpha$ reflects the relative penalty on violating the soft specification.
%In the following we show that for any accepting run $R_k$ of $\tilde{\mathcal{A}}_{p,k}$ its projection on $\pi$ $\tau_k = R_k|_{\Pi}$ is both valid and safe for $\mathcal{T}_k$ and $\varphi_k^{\text{hard}}$.
%===========================
%==========================
\begin{theorem}\label{ch03:theo:safety}
Assume $R$ is an accepting run of $\tilde{\mathcal{A}}_{r}$. Its projection on $\Pi$, $\tau=R|_{\Pi}$, is both \textbf{valid} and \textbf{safe} for $\mathcal{T}_c$ and $\varphi$ by Definition~\ref{ch03:def:vs}.
\end{theorem}
\begin{proof}
Firstly, $\tau$ is valid since every transition in $\delta'$ corresponds to a valid transition within~$\longrightarrow_c$. Then since $R$ is an accepting run of $\tilde{\mathcal{A}}_{r}=\mathcal{T}_c  {\times} \tilde{\mathcal{A}}_{\varphi}$, then $\texttt{trace}(\tau)\in \mathcal{L}_{\omega}(\tilde{\mathcal{A}}_{\varphi})$, i.e., $\texttt{trace}(\tau)\in \mathcal{L}_{\omega}({\mathcal{A}}^{\text{hard}})$ by Theorem~\ref{ch03:theo:saferun}. Since $\texttt{Words}(\varphi^{\text{hard}})= \mathcal{L}_{\omega}({\mathcal{A}}^{\text{hard}})$ by Lemma~\ref{ch02:lem:LTL-BA},  $\texttt{trace}(\tau)\in \texttt{Words}(\varphi^{\text{hard}})$, i.e.,~$\tau \models \varphi^{\text{hard}}$, thus $\tau$ is  safe.\end{proof}

\begin{split}

the accumulated distance of $\tau$ with respect to $\mathcal{A}_{\varphi^{\text{soft}}}$ is
$\texttt{dist}_{\varphi^{\text{soft}}}=(\sum_{i=0}^{f-1} \, +\, \gamma \sum_{i=f}^{n-1})$ $\texttt{Dist}(L_c(\pi_i),\, \chi_2(q'_{i}|_{Q_2},q'_{i+1}|_{Q_2})),$ where  $q'_{i}|_{Q_2}$ and $q'_{i+1}|_{Q_2}$ are the projection of $q'_i$ and $q'_{i+1}$ onto $Q_2$.

\begin{problem}\label{ch03:pro:safe}
Find the accepting run of $\tilde{\mathcal{A}}_{r}$ that \textbf{minimizes} the cost by~\eqref{ch03:eq:prod-cost3}. \hfill $\blacktriangle$
\end{problem}

We call the solution to Problem~\ref{ch03:pro:safe} as the \emph{safe} accepting run of $\tilde{\mathcal{A}}_{r}$, denoted by $R_{\textup{safe}}$. The corresponding \emph{safe} plan is $\tau_{\textup{safe}}=R_{\textup{safe}}|_{\Pi}$.


\subsubsection{Safe plan synthesis}

Given the values of $\alpha$ and $\gamma$,  $\tilde{\mathcal{A}}_{r}$ can be either constructed fully by Definition~\ref{ch03:def:relaxpro} or on-the-fly construction. But now the distance $d$ reflects whether the input alphabet violates the hard specification and the distance to the set of input alphabets for the soft specification. Then the safe accepting run $R_{\text{safe}}$ can be obtained in the prefix-suffix format by calling Algorithm~\ref{ch02:alg:optimal} with respect to $\tilde{\mathcal{A}}_{r}$. By Theorem~\ref{ch03:theo:safety}, its corresponding plan $\tau_{\text{safe}}$ is always valid and safe no matter how $\alpha$ and $\gamma$ are chosen.


Same as in Section~\ref{ch03:sec:infeasible}, the value of $\alpha$ could be tuned by calling Algorithm~\ref{ch02:alg:optimal} under different $\alpha$ to generate candidates of $R_{\textup{safe}}$. The associated $\texttt{cost}_{\tau}$ and $\texttt{dist}_{\varphi^{\textup{soft}}}$ can be computed similarly. After deciding the safe accepting run, its corresponding plan $\tau_{\textup{safe}}$ can be implemented by the hybrid control strategy following Algorithm~\ref{ch02:alg:hybrid}.


\begin{lemma}\label{ch03:lem:distanceProofsafe}
If $\textup{\texttt{dist}}_{\varphi^{\textup{soft}}}=0$, then $\tau_{\textup{safe}} \models \varphi$.
\end{lemma}
\begin{proof}
By Lemma~\ref{ch03:lem:distanceProof}, if $\textup{\texttt{dist}}_{\varphi^{\text{soft}}}=0$, $R_{\text{safe}}$ is an accepting run for the un-relaxed product $\mathcal{T}_c\times \mathcal{A}_{\varphi}$ by Definition~\ref{ch03:def:relaxpro}, where $\mathcal{A}_{\varphi}$ is the un-relaxed intersection~\citep{baier2008principles} of $\mathcal{A}_{\varphi^{\textup{soft}}}$ and $\mathcal{A}_{\varphi^{\textup{hard}}}$. Thus its corresponding plan $\tau_{\textup{safe}}$ satisfies $\varphi$ by Lemma~\ref{ch02:lem:satisfy}.
\end{proof}

\subsection{On-line plan adaptation}\label{ch03:sec:partial}
It is rarely the case that the system model, i.e., the transition system, is consistent with the actual workspace and robot dynamics. It means that the motion and task plan synthesized off-line may not be executed as expected. As a result, a real-time on-line control framework is needed, where the planning and execution are interleaved as shown in Figure~\ref{ch03:fig:online}.  If the actual workspace is different from the workspace model, it is crucial to put the planner on-line and make it dynamic, such that it can monitor the plan execution, update the system model based on the real-time observation, and validate or revise the current plan.

%%%%%%%%%%%%%%%%%%%
\begin{figure}[t]
\centering
\includegraphics[height =6cm]{figures/ch03/on-line-plan.pdf}
\caption{Diagram for the on-line planning and execution module.}
\label{ch03:fig:online}
\end{figure}

Thus we discuss here how the FTS could be updated based on the robot's observations that come from both its sensing or communication functionality.  We denoted by $\mathcal{T}_c^t$ the transition system at time $t\geq 0$, particularly
\begin{equation}\label{ch03:eq:t-FTS}
 \mathcal{T}_c^t=(\Pi,\, \longrightarrow_c^t,\, \Pi_0,\,AP,\, L_c^t,\, W_c^t),
\end{equation}
where the superscript indicates the time. Note that (i) the set of regions $\Pi$ is static, meaning that no new regions are added or existing regions are removed; (ii) the set of initial states $\Pi_0$ and the set of known atomic propositions $AP$ are also static as before. Furthermore, its task specification consists of hard and soft parts:
$
\varphi=\varphi^{\text{soft}} \, \wedge \, \varphi^{\text{hard}},
$
as introduced in~\eqref{ch03:eq:spec}, which is invariant after the system starts. Denote by $\tilde{\mathcal{A}}_{\varphi}$ the relaxed intersection automaton from Algorithm~\ref{ch03:alg:relaxedint}.

\begin{problem}
Assume the workspace is partially-known. The following problems are posed: (i) how to \textbf{model} the robot's sensing and communication functionalities; (ii) how to \textbf{update} the transition system accordingly; (iii) how to guarantee the motion and task plan is always \textbf{valid and safe}. \hfill $\blacktriangle$
\end{problem}


\subsubsection{Initial plan synthesis}

At $t=0$, the initial motion and task plan can be obtained by the method proposed in Section~\ref{ch03:sec:soft-hard}, given the initial transition system $\mathcal{T}_c^0$ and $\varphi$.  Denote by $R^t$, $\tau^t=R^t|_{\Pi}$, $\tilde{\mathcal{A}}_r^t$ the obtained safe accepting run, the safe plan and the relaxed product automaton at time $t\geq 0$. In this section, we assume that $\tilde{\mathcal{A}}_r^t$ is constructed on-the-fly and $R^t$ is obtained by Algorithm~\ref{ch02:alg:optimal} for $\tilde{\mathcal{A}}_r^t$. Note that the soft specification may be infeasible initially but it is guaranteed by Theorem~\ref{ch03:theo:safety} that $\tau^0$ is safe and valid for $\mathcal{T}_c^0$.


%=============
\subsubsection{Real-time knowledge update}\label{ch03:subsec:knowledge-update}

The robot we consider has both the sensing ability to discover the workspace and the communication functionality to communicate with external sources. In this section, we discuss how these functionalities can be modeled.

Denote by $\textbf{Sense}^t$ as the set of sensing information obtained at time $t\geq 0$. Note that this information might be gathered when a robot reaches a region or during the transition from one region to another. It has the following format:
\begin{equation}\label{ch03:eq:sense-info}
\textbf{Sense}^t =  \{\left((\pi,\,S,\,S_{\neg}),\, E,\, E_{\neg}\right)\},
\end{equation}
where $\pi\in \Pi$ stands for a perceived region; $S\subseteq {AP}$ is the set of propositions satisfied by this region; $S_{\neg}\subseteq{AP}$ is the set of propositions not satisfied by this region; $(\pi_i,\, \pi_j,\, w)\in E$ if $(\pi_i,\, \pi_j)$ needs to be added to $\longrightarrow^t_c$ with weight $w$ or its weight is updated to $w$;  $(\pi_i,\, \pi_j)\in E_{\neg}$ if $(\pi_i,\, \pi_j)$ needs to be removed from $\longrightarrow_c^t$.  $\textbf{Sense}^t$ reflects the actual workspace  at time~$t$.

\begin{example}\label{ch03:examp:sense}
The sensing info $\left((\pi_1, \{a_1,\, a_3\}, \{a_2\}),\, (\pi_1,\, \pi_2,\, 10), \, (\pi_1, \, \pi_3)\right)$ $\in \textup{\textbf{Sense}}^t$ is received if it is observed that region $\pi_1$ satisfies proposition $a_1$  and $a_3$ but not $a_2$; $(\pi_1,\, \pi_2,\, 10)\in E$ if the transition from  $\pi_1$ to $\pi_2$ is allowed with the cost $10$; $(\pi_1, \, \pi_3)\in E_{\neg}$ if the transition from  $\pi_1$ to $\pi_3$ is invalid.  \hfill $\blacktriangle$
\end{example}

This sensing function can be modeled by assigning a sensing radius $h>0$, such that all regions intersecting with the sphere $\{y \in \mathbb{R}^n\,|\,|y-x(t)|\leq h\}$ are visible, where $x(t)\in \mathbb{R}^n$ is the robot's position at time $t$. Different post-processing techniques might be used to abstract the essential information for~\eqref{ch03:eq:sense-info} from  raw sensing data.

Besides, communication with external sources is another important mean to retrieve information. This source can be another robot, a control base station, or even an on-line database.
Whenever this robot communicates with the external source at time $t$, it sends the following request message:
\begin{equation}\label{ch03:eq:request-info}
\textbf{Request}^t= \varphi|_{AP}\, ,
\end{equation}
which informs the external  source the set of workspace properties this robot is interested in. The reply message it gets has the following format:
\begin{equation}\label{ch03:eq:reply-info}
\textbf{Reply}^t= \{(\pi',\,S',\,S'_{\neg})\},
\end{equation}
where $ S' \subseteq (\varphi|_{AP})$ and $ S'_{\neg} \subseteq (\varphi|_{AP})$; $S'$ and $S'_{\neg}$ can not both be empty;  $\pi'\in \Pi$ is the region that satisfies $S'$ but not $S'_{\neg}$. Note that $S'$ and $S'_{\neg}$ only contain propositions that are relevant to the task $\varphi$. Depending on the actual communication protocol, the external source can decide how often it replies to the robot's request.

\begin{example}\label{ch03:examp:commu}
The reply information $(\pi_1,\, \{a_1\},\, \{a_2\})\in \textup{\textbf{Reply}}^t$ is received if region $\pi_1$ satisfies proposition $a_1$ but not $a_2$.  \hfill $\blacktriangle$
\end{example}


%====================
\subsubsection{Transition system update}\label{ch03:subsubsec:updateT}


Thus at time $t$, the robot might obtain new knowledge from $\textbf{Sense}^t$ and $\textbf{Reply}^t$ as described before, based on which it needs to update its own system model. \subsubsection{Verifying validity and safety}\label{ch03:subsec:revising}
Recall that the current accepting run~$R^t$ has a finite set of transitions appearing in it, i.e.,~$\texttt{Edge}(R^{t})$ by~\eqref{ch02:edgeR}. By checking~$\texttt{Edge}(R^{t})$, we could validate if the current plan~$\tau^t$ is still valid or safe.
%Denote by $\textup{\texttt{Edge}}(R^{t^-})$ the set of transitions appearing in~$R^{t^-}$, which is finite due to the prefix-suffix structure by~\eqref{ch03:eq:prefix-suffix}.

\begin{definition}\label{ch03:def:vsTS}
Given the updated transition system $\mathcal{T}_c^{t^+}$, any transition $(\langle\pi_i,\,q_m\rangle,$ $\,\langle\pi_j,\,q_n\rangle)\in \textup{\texttt{Edge}}(R^{t})$ is called: (i)  \textbf{invalid} if $(\pi_i,\, \pi_{j})\notin \longrightarrow_c^{t^+}$; (ii) \textbf{unsafe} if $L_c^{t^+}(\pi_i)\notin \chi_1(q_m|_{Q_1},\, q_n|_{Q_1})$.\hfill $\blacktriangle$
\end{definition}
Recall that~$Q_1$ is the set of states of $\mathcal{A}^{\textup{hard}}$.
We use~$\Xi$ and~$\aleph$ to store the invalid and unsafe transitions in $R^{t}$, respectively.
They are obtained by iterating through each transition  $(\langle\pi_i,\,q_m\rangle,\,\langle\pi_j,\,q_n\rangle)$ within $\texttt{Edge}(R^{t})$ and checks if $(\pi_i,\,\pi_j)$ has been removed from $\mathcal{T}_c^{t^+}$ or if the changed label $L_c^{t^+}(\pi_i)$ would make this transition unsafe.

\begin{theorem}\label{ch03:theo:validity}
Assume $R^{t}$ is an accepting run of $\tilde{\mathcal{A}}_{r}^{t^-}$; $\mathcal{T}_c^{t^-}$ is updated to $\mathcal{T}_c^{t^+}$; $\Xi$, $\aleph$ are the sets of invalid and unsafe transitions from above. Then (i) $\tau^{t}$ remains \textbf{valid} if and only if $\Xi =\emptyset$; (ii) $\tau^{t}$ remains \textbf{safe} if $\aleph=\emptyset$.
\end{theorem}
\begin{proof}
Since  $R^{t}$ is an accepting run of $\tilde{\mathcal{A}}_{r}^{t^-}$, $\tau^{t}$ is both valid and safe for $\mathcal{T}_c^{t^-}$ by Theorem~\ref{ch03:theo:safety}. If~$\Xi =\emptyset$ and $\aleph =\emptyset$, $\texttt{Edge}(R^{t})$ does not contain any invalid or unsafe transitions. Thus $R^{t}$ remains an accepting run of $\tilde{\mathcal{A}}_{r}^{t^+}$. ``If'' part of (i): by Theorem~\ref{ch03:theo:safety}, $\tau^{t}$ is valid since $R^{t}$ remains an accepting run of $\tilde{\mathcal{A}}_{r}^{t^+}$. ``Only if'' part of (i): if $\Xi \neq \emptyset$, $\tau^{t}$ contains at least one invalid transition, thus not valid by Definition~\ref{ch03:def:vs}. ``If'' part of (ii):~by Theorem~\ref{ch03:theo:safety}, $\tau^{t}$ is safe since $R^{t}$ remains an accepting run of $\tilde{\mathcal{A}}_{r}^{t^+}$.
\end{proof}

A plan revision scheme is needed to guarantee its validity and safety: one straightforward approach could be to  recall Algorithm~\ref{ch02:alg:optimal} with respect to $\mathcal{T}_c^{t^+}$ and $\mathcal{A}_\varphi$, but using the robot's current region $\pi_{\textup{cur}}$ from Algorithm~\ref{ch02:alg:hybrid} as the initial region. Given the robot's past trajectory $\tau_{\textup{past}}$ and past run $R_{\textup{past}}$  from Algorithm~\ref{ch02:alg:hybrid}, its complete trajectory is obtained by {concatenating} $\tau_{\textup{past}}$ with $\tau_{\textup{new}}$, namely

A key observation is that the safety property of $\tau_{\textup{new}}$ does not ensure the safety of the robot's complete trajectory starting from time $0$. \begin{problem}\label{ch03:prob:concatenate}
Assume $\Xi$ or $\aleph$ are not empty. Given the robot's past trajectory $\tau_{\textup{past}}$, how to find the \textbf{new plan} $\tau_{\textup{new}}$ such that its complete trajectory above is guaranteed to be \textbf{valid and safe}.  \hfill $\blacktriangle$
\end{problem}

Before stating the solution, we need to define the set of corresponding product runs given the robot's past trajectory. As pointed out in Definition~\ref{ch02:def:cor-run}, the resulting run of an infinite or finite words in $\mathcal{A}_{\varphi}$ may not be \emph{unique} because of the non-determinism of $\tilde{\mathcal{A}}_{\varphi}$. In other words, given the finite past trajectory $\tau_{\textup{past}}$, its trace may result in a set of runs in $\tilde{\mathcal{A}}_{\varphi}$, denoted by $r_{\tau_{\textup{past}}}$:
\begin{equation}\label{ch03:eq:buchi-past}
r_{\tau_{\textup{past}}}=\{r_{\bar{\sigma}}\; \text{by Def.~\ref{ch02:def:cor-run} for}\,\tilde{\mathcal{A}}_{\varphi}\,|\,\bar{\sigma}=\texttt{trace}(\bar{\tau}_{\textup{past}})\},
\end{equation}
where $\bar{\tau}_{\textup{past}}=\tau_{\textup{past}}[1:(|\tau_{\textup{past}}|-1)]$, i.e., the segment from the first state to the second last state of $\tau_{\textup{past}}$; $r_{\tau_{\textup{past}}}$ is finite because $\tau_{\textup{past}}$ is finite and the number of states in $\tilde{\mathcal{A}}_{\varphi}$ is finite.
Consequently, the finite set of corresponding  runs in $\tilde{\mathcal{A}}_{r}^{t^+}$ is given by
\begin{equation}\label{ch03:eq:prod-past}
R_{\tau_{\textup{past}}}=\{R\,|\, R|_{\Pi}=\tau_{\textup{past}},\; R|_{Q}\in r_{\tau_{\textup{past}}}\},
\end{equation}
where each finite run $R$ is the synchronized product of $\tau_{\textup{past}}$ and any run belonging to $r_{\tau_{\textup{past}}}$ by~\eqref{ch03:eq:buchi-past}. Since $R_{\tau_{\textup{past}}}$ may contain multiple finite runs in $\tilde{\mathcal{A}}_{r}^{t^+}$, $\pi_{\textup{cur}}$ may correspond to multiple states in $\tilde{\mathcal{A}}_{r}^{t^+}$, which is denoted by
\begin{equation}\label{ch03:eq:set-past}
Q'_{\tau_{\textup{past}}}=\{\texttt{last}(R)\,|\, R\in R_{\tau_{\textup{past}}}\},
\end{equation}
where $\texttt{last}(R)$ is the last state of $R$ within $R_{\tau_{\textup{past}}}$. Clearly, $R_{\text{past}}\in R_{\tau_{\textup{past}}}$ and $q'_{\textup{cur}}\in Q'_{\tau_{\textup{past}}}$; $Q'_{\tau_{\textup{past}}}$ is derived as the reachable states in $\tilde{\mathcal{A}}_{r}^{t^+}$ given $\tau_{\textup{past}}$.

To solve Problem~\ref{ch03:prob:concatenate}, we propose an algorithm that is similar to Algorithm~5~in~\cite{guo2013revising}. Denote by $R^{t^-}$, $R^{t^+}$, $\tau^{t^-}$, $\tau^{t^+}$ the accepting run and corresponding plan before and after the revision, respectively. Moreover, $R_{\text{pre}}^{t^-}$ and $R_{\text{suf}}^{t^-}$ are the prefix and suffix of $R^{t^-}$, while $R_{\text{pre}}^{t^+}$ and $R_{\text{suf}}^{t^+}$ are the prefix and suffix of $R^{t^+}$.
Given the invalid or unsafe transition $(q'_s,\,q'_{s+1})$ in $R^{t^-}$, it belongs to either $R^{t^-}_{\text{pre}}$ or $R^{t^-}_{\text{suf}}$. In Algorithm~\ref{ch03:alg:revise}, it firstly tries to locally finds a ``bridging'' segment that make up this transition by breadth-first search (as shown in Figure~\ref{ch03:fig:tworevise}).  $\texttt{tail}(q'_{s+1}, R^{t^-}_{\text{pre}})$ is  the segment of $R^{t^-}_{\text{pre}}$ after $q'_{s}$, not including $q'_{s}$; $\texttt{head}(q'_{s-1}, R^{t^-}_{\text{pre}})$ is the segment of  $R^{t^-}_{\text{pre}}$ before $q'_{s-1}$, not including $q'_{s-1}$; $\texttt{last}(bridge)$ is the last state on the path $bridge$. Function \texttt{DijksTarget}($\tilde{\mathcal{A}}_{r}^{t^+}$, $q'_S$, $Q'_T$)  is defined similarly as function \texttt{DijksTargets}($\cdot$) in~Algorithm~\ref{ch02:alg:optimal}, which returns shortest path from the source state $q'_S\in Q'$ to \emph{any} target state belonging to the set $Q'_T\subseteq Q'$. Thus it returns the shortest path once one of the targets is reached. At last, if function \texttt{DijksTarget}($\cdot$) returns an empty path $bridge$, it means that the accepting state $\textup{\texttt{last}}(R_{\textup{pre}}^{t^-})$ is not reachable from $q'_{s-1}$. Then $\texttt{OptRun}(\cdot)$ from Algorithm~\ref{ch02:alg:optimal} is called to search for the balanced accepting run of~$\tilde{\mathcal{A}}_{r}^{t^+}$, but using $Q'_{\tau_{\textup{past}}}$  as the set of initial states (instead of the default $Q'_0$). Note that in Algorithm~\ref{ch03:alg:revise} above, $R^{t^-}$ is revised iteratively and the condition $(q'_s,\,q'_{s+1})\in R^{t^-}_{\text{pre}}$ or $R^{t^-}_{\text{suf}}$ is also checked iteratively.

\begin{theorem} \label{ch03:theo:new-safe}
The new plan $\tau_{\textup{new}}$ can be obtained from Algorithm~\ref{ch03:alg:revise} such that $\tau_{\textup{comp}}$ is \textbf{valid and safe}, if there exists one.
%a valid and safe path $\tau_{\textup{comp}}$ by Definition~\ref{ch03:def:vs}, if there exists an infinite path
%Given $\tau_{\textup{past}}$ and $\tilde{\mathcal{A}}_{r}^{t^+}$,
\end{theorem}
\begin{proof}
If a new plan $\tau_{\textup{new}}$ exists such that the complete path $\tau_{\textup{comp}}$ is valid and safe, by Lemma~\ref{ch02:lem:non-empty} there exists an accepting run of $\tilde{\mathcal{A}}_{r}^{t^+}$ whose projection onto $\Pi$ is $\tau_{\textup{comp}}$. Given the invalid or unsafe transitions in $R^{t^-}$ (in prefix or suffix), firstly Algorithm~\ref{ch03:alg:revise} tries to revise $R^{t^-}$ by looking for the bridging segments. If no such segments exist, it means that the current accepting state is not reachable from $q_{s-1}'$. Instead it searches for the accepting run that starts from any of the product states in $Q'_{\tau_{\textup{past}}}$ and consists of a cycle containing at least one of accepting states in $\tilde{\mathcal{A}}_{r}^{t^+}$. It means that an accepting run that starts from one of the initial states and \emph{obeys} the robot's past trajectory exists only if a bridging segment is found for the revision in Line~3  or a new accepting run starting from $Q'_{\tau_{\textup{past}}}$ is found in Line~8.
\end{proof}

The overall structure of the on-line planning scheme is given in Algorithm~\ref{ch03:alg:scheme} later, where the accepting run~$R^t$ is updated by both the revision Algorithm~\ref{ch03:alg:revise} and the optimal search Algorithm~\ref{ch02:alg:optimal}.
More details about the algorithms can be found in~\cite{guo2013cooperative}.

\subsection{Collaborative knowledge transfer}\label{ch03:sec:multi-partial}

Now assume that \emph{a team} of autonomous robots coexist within the same workspace and each of them has an individual local task specified as LTL formulas. If the local task of each robot  relies on only its local propositions, the team of robots are independent that their plans can be synthesized and executed independently from each other. However since the robots are usually located at different locations within the workspace, about which they have up-to-date knowledge.  Those local knowledge, if shared among the team, might benefit each member such that they can make better and more informed plans. In the following, we propose a collaborative knowledge transfer scheme for a multi-robot system with independent local tasks.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%\subsubsection{Multi-robot systems}\label{ch03:subsec:local-tasks}
Formally, we consider a team of autonomous robots with unique  identities~$i\in \mathcal{N}=\{1,2,\cdots,N\}$, which coexist within the common but partially-known workspace~$\mathcal{W}$. Robot $i$'s possible motion within $\mathcal{W}$ is abstracted as a weighted FTS:
\begin{equation}\label{ch03:eq:Tk}
\mathcal{T}_i^t= ({\Pi}_{i},\, \longrightarrow_{i}^t, \, \Pi_{i,0},\,  AP_{i}, \,L_{i}^t ,\, W_{i}^t),
\end{equation}
which is defined similarly  as~\eqref{ch03:eq:t-FTS}. The superscript $t\geq 0$ indicates that the workspace is partially known and might be updated after the system starts. Each robot $i\in \mathcal{N}$ has a locally-assigned task specification $
\varphi_i=\varphi_{i}^{\text{soft}} \, \wedge \, \varphi_{i}^{\text{hard}}$, where $\varphi_i^{\text{soft}}$ and $\varphi_i^{\text{hard}}$ are the ``soft'' and ``hard'' constraints as before from~\eqref{ch03:eq:spec}.

\subsubsection{Initial synthesis}
At $t=0$, for each robot $i\in \mathcal{N}$, the initial motion and task plan can be obtained as the safe plan by following the framework proposed in Section~\ref{ch03:sec:soft-hard}, regarding the initial transition system $\mathcal{T}_i^0$ and $\varphi_i$.  We assume here the on-the-fly construction is used. Note that the soft specification may not be feasible initially and is relaxed by the balanced accepting run. Denote by $\tilde{\mathcal{A}}_{r,i}^t$ the relaxed product automaton of robot~$i$ at time~$t$; $R_i^t$, $\tau_i^t$ as the balanced accepting run and the associated plan.

\subsubsection{Knowledge transfer protocol}\label{ch03:subsec:know-transfer}
In Section~\ref{ch03:sec:partial}, we describe how a single robot could update its transition system through its sensing and communication ability to inquire and retrieve knowledge from external sources.  Belonging to the same multi-robot system, the other robots could be the external source. In other words, they could share and transfer their knowledge about the workspace collectively in real-time. Now we explain how to design the knowledge transfer protocol and more importantly how to combine it with the real-time  planning scheme from Section~\ref{ch03:sec:partial}.

%==================

The communication network represents how the information flows among the robots. Each robot $i$ has a set of neighboring robots, denoted by $\mathcal{N}_i \subseteq \mathcal{N}$. Robot $i$ can send messages directly to any robot belonging to $\mathcal{N}_i$. We take into account two different ways to model the communication network: (1) global communication with a fixed topology; (2) limited communication with a dynamic topology. In the first case, $\mathcal{N}_i$ is pre-defined and fixed after the system starts. In the second case, each robot has a communication range, denoted by $C_i\geq 0$. Robot $i$ can only send messages to robot $j$ if their relative distance is less than $C_i$, i.e., $|x_j(t)-x_i(t)|\leq C_i$ where $x_j(t)$, $x_i(t)\in \mathbb{R}^n$ are the positions of robots $j$ and $i$ at time $t$. Then $\mathcal{N}_i^t=\{j\in \mathcal{N}|\,|x_j(t)-x_i(t)|\leq C_i \}$ is  the time-varying neighboring set of robot $i$ at time $t$.

\begin{algorithm}[t]
\caption{Transfer Knowledge to other robots, \texttt{TranKnow}(~)} \label{ch03:alg:transferAlgo}
%\linesnumbered
%\dontprintsemicolon
%\LinesNumbered
\DontPrintSemicolon
\KwIn{$\textbf{Sense}_i^t$, $\textup{\textbf{Request}}_{j,i}^{t_0}$}
\KwOut{$\textbf{Reply}_{i,j}^t$}
\ForAll{${(\pi,\,S,\,S_{\neg})}\in \textup{\textbf{Sense}}_i^t$}{
\ForAll{$\textup{\textbf{Request}}_{j,i}^{t_0}$ \textup{received at} $t_0<t$}{
  $\textbf{Request}_{j,i}^t= (j,\, \varphi_j|_{AP_j},\, i)$\;

 \If{$j\in \mathcal{N}_i^t$}{
$S' = S\,\cap\, (\varphi_j|_{AP_j})$,\;
$S'_{\neg} = S_{\neg}\,\cap\, (\varphi_j|_{AP_j})$\;

\If{$S'\neq \emptyset$ \textup{or} $S'_{\neg}\neq \emptyset$}{
add $(\pi,\,S',\,S'_{\neg})$ to $\textbf{Reply}_{i,j}^t$}
}
}
}
\textbf{return} $\textbf{Reply}_{i,j}^t$
\end{algorithm}

Robot~$i$ is  {interested in} all the propositions appearing in   $\varphi_i$, namely  $\varphi_i|_{AP_i}$. We propose a \emph{subscriber-publisher}  mechanism to reduce the communication load. Whenever robot $j$ communicates with robot $i\in \mathcal{N}^t_j$ for the \emph{first} time at time $t$, it follows the \emph{subscribing} procedure: robot $j$ sends a request message to robot $i$:
\begin{equation}\label{ch03:eq:robot-request}
\textbf{Request}_{j,i}^t= (j,\, \varphi_j|_{AP_j},\, i),
\end{equation}
which informs robot $i$ the set of propositions robot $j$ is interested. Each robot has a subscriber list, containing the request messages it has received. Note that each robot also keeps track of the robots which it has subscribed to, such that it sends a request to any of its neighboring robots only once.

The sensing update of robot $i$ is denoted by $\textbf{Sense}_i^t$, the structure of which is given in~\eqref{ch03:eq:sense-info}. Then the \emph{publishing} phase of each robot follows an event-driven approach: whenever $(\pi,\,S,\,S_{\neg})\in \textbf{Sense}_i^t$ is obtained, it checks its subscriber list  whether the content might be  of interest to any of the subscribers regarding some propositions. If it is of interest to robot $j$ regarding some propositions in  in $\varphi_j|_{AP_j}$, then robot $i$ checks if $j\in \mathcal{N}_i^t$. If so, it publishes a reply message to robot $j$ that
\begin{equation}\label{ch03:eq:robot-reply}
\textbf{Reply}_{i,j}^t= \{(\pi,\,S',\,S'_{\neg})\},
\end{equation}
where $ S' = S\cap (\varphi_j|_{AP_j})$ and $ S'_{\neg} = S_{\neg}\cap (\varphi_j|_{AP_j})$; Note that since $S'$ and $S'_{\neg}$ only contain propositions that are relevant to robot's task $\varphi$, every reply message should contain useful knowledge.  The above procedure is summarized in Algorithm~\ref{ch03:alg:transferAlgo}. Note that through this communication mechanism, any request only needs to be sent once and every reply message contains useful knowledge. This subscriber-publisher scheme can be easily implemented by  multicast or unicast wireless protocols.

Regarding its current motion and task plan $\tau_i^t$, the validity and safety of $\tau_i^t$ needs to be verified as before. Algorithm~\ref{ch03:alg:revise} provides a way to locally revise the invalid or unsafe plan, which  guarantees validity and safety by Theorem~\ref{ch03:theo:safety}. However it does not maintain the cost optimality of the safe accepting run compared with Algorithm~\ref{ch02:alg:optimal}. The general problem of computing and maintaining the shortest path in a graph where the edges are inserted or deleted and edge weights are increased or decreased is referred to as the \emph{dynamic shortest path problem}~(DSPP) in~\cite{chan2009shortest} and~\cite{misra2005dynamic}. As  pointed out in both papers, it is inefficient to re-compute the shortest path ``from scratch''  using the well-known  {static} solution like Dijkstra algorithm each time the graph changes.
%Meanwhile, the proposed DSPP algorithms   require much more complicated mechanism and sometimes not more efficient than calculating everything from scratch.

Thus we propose an event-based criterion~\cite{heemels2012introduction} to ensure the optimality check. Denote by $\Upsilon^{t}_i$ the accumulated number of number of changes in $\mathcal{T}_i^t$ at time $t$; $\Upsilon^{t^+}_i=\Upsilon^{t^-}_i\,+\, |\widehat{\Pi}_{i}^t|$ and $\Upsilon^{0}_i=0$. Denote by $T_i$ the last time instant when Algorithm~\ref{ch02:alg:optimal} is called. Let the  thresholds $N^{\text{call}}_i,\, T_i^{\text{call}}\geq 0$ be chosen freely by robot~$i\in \mathcal{N}$. Then whenever one of the conditions holds: (1) $\Upsilon^{t}_i\geq N_i^{\text{call}}$; (2) $t-T_i\geq T_i^{\text{call}}$, Algorithm~\ref{ch02:alg:optimal} is called with respect the latest $\mathcal{T}_i^t$ to derive the safe plan, but using $Q'_{\tau_{\textup{past}}}$ as the initial states. Afterwards, $\Upsilon^{t^+}_i$ is reset to zero and  $T_i$ to the current time.

\subsubsection{Overall structure}\label{ch03:subsec:structure}

The overall architecture is summarized in Algorithm~\ref{ch03:alg:scheme}, where the detail description of each function can be found in~\cite{guo2014lic}. When the system starts, each robot synthesizes its own initial motion and task plan. It sends requests to neighboring robots. Then it checks if it receives any reply, sensing or request messages, based on which it replies to its subscribers, and updates its transition system. At last, it decides whether the revising algorithm or the optimal synthesis algorithm should be called by the triggering condition. It is worth mentioning that the next goal region $\pi_{i,\textup{next}}$ changes automatically whenever: (i) the accepting run $R_i^t$ is updated by either the revision Algorithm~\ref{ch03:alg:revise} and the optimal search Algorithm~\ref{ch02:alg:optimal}; (ii) the current goal region is confirmed to be reached.

\begin{theorem}\label{ch03:theo:improvement}
For each robot $i$ at any time $t\geq 0$, its plan $\tau_i^t$ derived by  Algorithm~\ref{ch03:alg:scheme} is always  \textbf{valid} and  \textbf{safe} for $\mathcal{T}_i^t$ and $\varphi_i$. Moreover, for any $t'\geq 0$, there exists time $t\in [t',\,t'+T^{\text{call}}_i]$ such that $\tau_i^t$ corresponds to the \textbf{safe} accepting run.
\end{theorem}
\begin{proof}
The first part follows  from Theorems~\ref{ch03:theo:validity} and~\ref{ch03:theo:new-safe}.  Moreover $\tau_i^t$ is the {safe} plan for $\tilde{\mathcal{A}}_{r,i}^{t^+}$ given $Q'_{i,\tau_{\textup{past}}}$ whenever Algorithm~\ref{ch02:alg:optimal} is called. Due to the triggering condition, it is called at least once within any time period of length~$T^{\text{call}}_i$. \end{proof}
